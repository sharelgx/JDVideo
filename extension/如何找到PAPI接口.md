# 如何找到PAPI接口请求

## 步骤：

### 1. 打开开发者工具 Network 标签
- 按 `F12` 打开开发者工具
- 点击 **Network** / **网络** 标签

### 2. 清空之前的请求记录
- 点击左上角的 🚫 清除按钮
- 或按 `Ctrl+L` / `Cmd+K`

### 3. 筛选请求
在 Network 标签顶部的筛选框输入以下关键词之一：
- `papi`
- `api`
- `explain`
- `download`
- `video`

### 4. 点击"解析列表"按钮
- 在插件弹窗中点击"解析列表"
- 观察 Network 标签是否有新的请求出现

### 5. 如果没看到API请求，尝试点击页面的"下载"按钮
- 在页面上找一个商品的"下载"按钮
- 手动点击一次
- 观察 Network 标签是否出现新的请求

### 6. 查看请求详情
找到请求后（通常在 **XHR** 或 **Fetch** 分类下），点击它：

**查看 Request（请求）标签：**
- **Request URL**: 完整的请求地址（复制给我）
- **Request Method**: GET 或 POST
- **Query String Parameters**: 请求参数（如果有）
- **Request Payload**: POST 请求的请求体（如果有）

**查看 Response（响应）标签：**
- 查看返回的 JSON 内容
- 查找包含 `.mp4`、`.m3u8` 或 `download` 的字段
- 复制整个 Response 内容给我

## 常见情况：

### 情况1：看到了 API 请求
如果看到了包含 `papi`、`api` 或 `explain` 的请求：
1. **点击该请求**
2. **复制 Request URL**（完整的URL，如：`https://xxx.jd.com/papi/xxx/xxx?id=123456`）
3. **切换到 Response 标签，复制响应内容**
4. 把这两样信息发给我

### 情况2：没看到 API 请求
可能的原因：
1. **接口还没有被调用** - 需要点击"下载"按钮才会触发
2. **接口名称不包含 papi** - 可能需要查看所有 XHR/Fetch 请求
3. **接口在页面加载时就调用了** - 需要刷新页面时就开始监控

解决方案：
- 在 Network 标签，**不要筛选**，查看所有请求
- 点击 **XHR** 或 **Fetch** 按钮，只看这两种类型的请求
- 刷新页面，观察页面加载时的请求

### 情况3：看到了但不确定是不是
如果看到了请求但不确定：
- 点击请求，查看 **Preview** 或 **Response** 标签
- 如果响应是 JSON 格式，查找是否有类似这些字段：
  - `downloadUrl`
  - `videoUrl`
  - `url`
  - `mp4Url`
  - `fileUrl`
- 或者查找包含 `.mp4`、`.m3u8` 的字符串
